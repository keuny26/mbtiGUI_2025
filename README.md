# ✨ MBTI 간이 검사 프로그램 (GUI 버전) 주요 변경사항 및 특징

## 1. 🚀 기술 스택 전환: Tkinter GUI 도입

기존의 텍스트 기반(CLI, Command Line Interface) 프로그램에서 **그래픽 사용자 인터페이스(GUI)** 방식으로 전환되었습니다.

| 구분 | 이전 (CLI) | 현재 (GUI) |
| :--- | :--- | :--- |
| **사용 기술** | Python의 `input()`을 활용한 **표준 입출력** | Python 내장 라이브러리 **`Tkinter`** 사용 |
| **프로그램 구조** | 순차적인 함수 호출 | **`MBTITestApp` 클래스** 기반의 객체 지향적 설계 |
| **상호작용** | 텍스트 입력 후 `Enter` | **버튼(Button) 클릭**을 통한 이벤트 처리 |

---

## 2. 🎨 사용자 경험(UX) 및 상호작용 개선

가장 큰 변화는 사용자가 프로그램을 사용하는 방식이 훨씬 직관적이고 편리해졌다는 점입니다.

### 2.1. 버튼 기반의 직관적 응답

사용자는 더 이상 키보드 입력을 통해 숫자를 치고 엔터를 누를 필요가 없습니다. 질문에 해당하는 **'매우 그렇다'** 또는 **'그렇지 않다' 버튼**을 클릭하여 즉시 응답합니다.

* **잘못된 입력** (예: 숫자 외 문자 입력)으로 인한 오류 발생 가능성이 **완전히 제거**되었습니다.

### 2.2. 질문 진행 상황 표시

화면 하단에 **`X / 12 진행 중`** 형태의 **진행 상황 레이블**을 추가하여 사용자가 현재 몇 번째 질문을 진행하고 있는지 명확하게 알 수 있습니다.

### 2.3. 시각적으로 분리된 결과 창

결과를 기존의 터미널 화면에 텍스트로 바로 출력하는 대신, **`Toplevel`**을 사용하여 **새로운 창**에 결과를 표시합니다.

* 최종 **MBTI 유형**을 큰 글씨와 강조 색상으로 표시하고, **지표별 해설** 및 **유형 요약**을 `Text` 위젯에 담아 시각적으로 깔끔하게 분리하여 보여줍니다.

---

## 3. ⚙️ 코드 구조의 특징

* **클래스 구조:** `MBTITestApp` 클래스를 중심으로 애플리케이션의 **UI 요소**와 **비즈니스 로직**을 통합하여 관리합니다. 이는 코드의 **유지보수성**과 **확장성**을 높여줍니다.
* **명시적 데이터 관리:** 질문과 해설 데이터를 프로그램 상단에 **상수 딕셔너리(Dictionary)** 형태로 명확하게 분리하여 관리합니다. 질문이나 해설 내용을 수정하기가 매우 용이합니다.
